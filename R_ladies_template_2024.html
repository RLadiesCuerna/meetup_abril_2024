Quarto es un sistema de publicación científica y técnica de código abierto que le permite combinar texto narrativo y código para producir resultados de alta calidad, incluidos informes, presentaciones, sitios web y más.

Científico y técnico: Significa que Quarto tiene funcionalidades específicas para artículos de revistas o trabajos científicos, como soporte para ejecución de código, citas, notas al pie, Markdown científico, ecuaciones, referencias cruzadas, paneles de figuras, leyendas, diseño avanzado y más. Veremos algunos de estos elementos más adelante. Código abierto: En RStudio creemos que es mejor para todos que las herramientas utilizadas en investigación y ciencia sean gratuitas y abiertas. El software libre en la ciencia significa mayor reproducibilidad, intercambio generalizado de conocimientos y técnicas, y eliminación de barreras de costo. También puedes ver el código fuente y contribuir. (ya que eres más experimentado)

Sistema de publicación: Con Quarto puedes crear contenido dinámico con Python, R, Julia y Observable. Quarto toma texto plano renderizado como archivos R Markdown o formatos mixtos como cuadernos Jupyter para generar artículos, informes, presentaciones, sitios web, blogs y libros de alta calidad en HTML, PDF, MS Word, ePub y más.

Diferencias entre Quarto y LaTeX Quarto y LaTeX son dos sistemas populares para la creación de documentos científicos y técnicos. Si bien ambos comparten algunas similitudes, también existen diferencias importantes a considerar al elegir uno para su proyecto.

TeX es un sistema de preparación de documentos creado por Donald E. Knuth. TeX no es un procesador de textos tradicional, sino que mas bien se parece a un lenguaje de programación. LATEX, paquete de macros creado por Leslie Lamport. dialectos de TEX(o mejor dicho distintos paquetes de macros)LaTeX es como un lenguaje de alto nivel para TEX. Similitudes:

Énfasis en la tipografía: Tanto Quarto como LaTeX se enfocan en la tipografía de alta calidad, lo que los hace ideales para documentos que requieren una presentación profesional. Formato flexible: Ambos sistemas ofrecen un alto grado de control sobre el formato del documento, desde el diseño de la página hasta la numeración de secciones y referencias. Soporte para código: Ambos admiten la inclusión de código fuente en el documento, con resaltado de sintaxis y otras características útiles. Diferencias:

Enfoque:

Quarto: Es un sistema de publicación completo que abarca desde la creación de documentos hasta la publicación en diferentes formatos (HTML, PDF, sitios web). LaTeX: Se enfoca principalmente en la composición tipográfica y la generación de documentos PDF. Facilidad de uso:

Quarto: Ofrece una interfaz gráfica de usuario (GUI) en RStudio, lo que facilita el aprendizaje y la creación de documentos para usuarios principiantes. LaTeX: Requiere el uso de código para definir el formato del documento, lo que puede tener una curva de aprendizaje más pronunciada. Lenguaje de marcado:

Quarto: Utiliza un lenguaje de marcado ligero y fácil de leer, similar a Markdown. LaTeX: Utiliza un lenguaje de marcado más complejo con comandos específicos para definir la estructura y el formato del documento. Integración con R:

Quarto: Se integra a la perfección con RStudio, lo que facilita la creación de documentos que combinan texto narrativo, código R y resultados. LaTeX: Requiere paquetes adicionales para la integración con R, como knitr

Interfaz de línea de comandos

A diferencia de R Markdown, Quarto no es un paquete de R. Quarto es una interfaz de línea de comandos. Esto hace que sea mucho más fácil trabajar con documentos Quarto fuera del IDE de RStudio. Por ejemplo, si alguna vez necesita renderizar un archivo desde un terminal, creemos que estará de acuerdo en que quarto render document.qmd es mucho más fácil de usar que: Para ver las otras opciones de la línea de comandos, ejecute quarto –help en la terminal. Quarto es un producto único la unificación del ecosistema R Markdown, así que profundicemos un poco más en lo que eso significa…

R Markdown isn’t going away but big new features will be in Quarto, things like YAML intelligence, figure panels, layouts, which we’ll cover later in the session.

Quarto es:

  • Es flexible y portable: Puedes usar Quarto para crear una amplia variedad de documentos. Los documentos Quarto se pueden generar en una variedad de formatos, como HTML, PDF, EPUB y DOCX.
  • Es fácil de aprender: Markdown es un lenguaje de marcado ligero y fácil de aprender.
  • Es gratis y de código abierto: Puedes usar Quarto de forma gratuita y el código fuente está disponible para que cualquiera lo pueda modificar.

Más importante

El autor tiene que preocuparse más por el contenido que por el formato.

En R Markdown, puede usar {distill} para crear un sitio web, {bookdown} para escribir un libro y {revealjs} para hacer diapositivas. Eso podría significar que está utilizando tres paquetes diferentes para crear tres tipos diferentes de contenido sobre el mismo tema. Quarto combina la funcionalidad de R Markdown, bookdown, distill y otros paquetes en un solo sistema. Esto puede facilitar la creación de contenido con múltiples formatos de salida. Por ejemplo, puede usar referencias cruzadas sin instalar {bookdown} y hacer presentaciones sin instalar {revealjs}. Tener menos dependencias hace que los proyectos sean más fáciles de mantener a largo plazo, ¡no tiene tantos paquetes R para actualizar!

Combinando contenido (y códigos abreviados)

Quarto le permite utilizar inclusiones al estilo de Hugo. Esto brinda una forma conveniente de reutilizar contenido en varios documentos. Por ejemplo, suponga que tiene un párrafo o dos que brindan una introducción a su empresa. En lugar de copiar y pegar constantemente, el párrafo se almacena en un documento separado, about.qmd, y se puede incluir usando {{< include about.qmd >}}. Agregar este fragmento a su documento es equivalent a copiar y pegar el texto de about.qmd en el archivo principal. La consecuencia es que los enlaces e imágenes relativos se resuelven según el directorio del documento principal.

include es solo un ejemplo de un código abreviado de Quarto. Otros códigos abreviados útiles a tener en cuenta son var y pagebreak. El código abreviado var le permite acceder a valores del archivo _variables.yml. En el pasado, es posible que haya escrito una función R corta para esta tarea. El código abreviado pagebreak proporciona un mecanismo conveniente para agregar saltos de página en diferentes formatos de salida, por ejemplo, HTML, PDF o doc.

También puede crear plantillas personalizadas para diferentes tipos de documentos. Por ejemplo, agregando la marca corporativa a un informe PDF.

BATERIAS INCLUIDAS

Una de las principales características de Quarto es que no solo está construido para R. Es independiente del lenguaje. Puede renderizar documentos que contienen código escrito en R, Python, Julia u Observable. Eso lo hace increíblemente útil si trabaja en equipos multilingües o colabora con personas que escriben en un lenguaje de programación diferente al suyo.
